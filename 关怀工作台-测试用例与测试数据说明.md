# 主动关怀工作台 — 测试用例与测试数据说明

> 配套测试数据：`xg-backend/xg-app/src/main/resources/db/migration/tenant/V131__seed_care_demo.sql`
> 覆盖范围：辅导员工作台 / 状态机 / SLA 超期 / 去重冷却 / AI brief / 院系·学校管理视图 / 规则运维 / 学生小程序只读 / 权限边界 / 通知联动 / 多租户隔离。

## 0. 文档说明

- **目的**：关怀工作台及配套功能在空库下"无数据可显"。本文档 = 一份可执行的人工 QA 测试用例集 + 一套幂等 demo 种子数据，让所有页面有真实数据，并系统性验证全功能面。
- **数据落地方式**：Flyway 租户迁移 `V131__seed_care_demo.sql`，沿用 `${tenant_id}` 占位 + `ON CONFLICT DO NOTHING` 幂等，启动自动应用，可重复执行不产生脏数据。
- **关键语义（务必先理解）**：
  - `care_task.student_id` = `sys_user.id`（学生用户 ID），与 `student_event_log.student_id` 同语义；**不是** `student_profile.id`。
  - `assigned_to` / `actor_id` / `submitted_by` = `sys_user.id`（辅导员/操作人）。
  - 规则集来自代码 `CareRuleCatalog`（版本 `p1-2026.05`），**不入库**；运维表只存"租户对内置规则的启停/偏移覆盖"。
  - 详情页"触发证据"= `care_task.trigger_data` JSONB 去掉 `rule_id`/`rule_version` 后的内容；**不读** `student_event_log`。
  - SLA 由 severity 派生：`critical=24h / high=48h / medium=7d / low=14d`（受全局严重度偏移整体移位）。

### B. 状态机迁移（CareTaskTransitions 合法矩阵）

合法矩阵：pending→{accept→accepted, reject→rejected, transfer→transferred, reschedule→pending, overdue_tick→overdue}；accepted→{save_followup→in_progress, resolve→resolved, reject→rejected, transfer→transferred, reschedule→pending, overdue_tick→overdue}；in_progress→{resolve→resolved, transfer→transferred}；overdue→{accept→accepted, resolve→resolved, transfer→transferred}；resolved/rejected/transferred = 终态无出边。

**TC-B01 接单 pending→accepted**
- 前置：13001 pending（李老师 2001）。
- 步骤：`POST /care/tasks/13001/accept`。
- 预期：status=accepted，accepted_at/accepted_by=2001 写入；`care_task_audit` 新增 accept 行(from=pending,to=accepted,actor=2001)。

**TC-B02 记录跟进 accepted→in_progress**
- 前置：13002 accepted。
- 步骤：`POST /care/tasks/13002/...`（保存跟进）。
- 预期：status=in_progress；审计 save_followup 行带 payload.note。

**TC-B03 完成 →resolved（任意非终态可达）**
- 前置：13003 in_progress。
- 步骤：`POST /care/tasks/13003/resolve` body `{"note":"已闭环"}`。
- 预期：status=resolved，closed_at/closed_by/closed_reason='resolved'；`cooldown_until = closed_at + R006.cooldown(14d)` 被物化。

**TC-B04 拒绝 pending→rejected + 必填原因**
- 前置：13006 pending。
- 步骤：`POST /care/tasks/13006/reject` body `{"reason_code":"student_special_case","reason_detail":"..."}`；再测缺 reason_code。
- 预期：合法→rejected，closed_reason=reason_code，cooldown 物化(R008=30d)；缺 reason_code → 400/业务异常，状态不变。

**TC-B05 改期 reschedule（pending/accepted/in_progress→pending，count+1）**
- 前置：13002（先 accept 到 accepted 或用原 accepted）。
- 步骤：`POST /care/tasks/13002/reschedule` body `{"days":3}`。
- 预期：status=pending，due_at 顺延 3 天，reschedule_count+1；审计 reschedule 行 payload `{"days":3}`。days 非 1/3/7 → 拒绝。

**TC-B06 转介 →transferred**
- 步骤：`POST /care/tasks/13002/transfer` body `{"target_dept":"aid_office","reason_detail":"..."}`。
- 预期：status=transferred，transferred_to=aid_office，closed_reason='transfer'，cooldown 物化。target_dept 非枚举(counseling_center/aid_office/academic_affairs/security) → 拒绝。

**TC-B07 非法迁移被拒**
- 前置：13004 resolved（终态）。
- 步骤：`POST /care/tasks/13004/accept`、`/resolve`。
- 预期：业务异常（非法状态迁移），状态不变，无审计写入。in_progress 调 accept 同样被拒。

**TC-B08 越权操作他人任务**
- 前置：13008（王老师 2002 的任务）。
- 步骤：李老师 2001 调 `POST /care/tasks/13008/accept`。
- 预期：requireOwnedTask 校验失败（assigned_to≠当前用户）→ 拒绝。
